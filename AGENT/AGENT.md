# 依頼：Visual Studio 2022 C++20 でそのままビルドできるオセロゲームの作成

以下の制約と仕様を完全に満たす、C++20のコンソール（CLI）オセロゲームのソースコードを作成してください。
出力の文字数制限に配慮し、コードが長い場合は「第1部」「第2部」「第3部」のように分割して、最終的にすべてのコードが合体可能な形で必ず最後まで出力してください。

---

## 1. 開発環境・前提条件
*   **開発環境**: Visual Studio 2022
*   **言語基準**: C++20
*   **プロジェクト種類**: C++ 空のプロジェクト、またはコンソールアプリ
*   **外部ライブラリ**: 使用禁止（C++標準ライブラリ `iostream`, `vector`, `string`, `conio.h` などの標準ヘッダーのみを使用すること）
*   **構成**: ヘッダーファイルとソースファイルを分けず、1つの `main.cpp` にすべてを記述し、コピペだけでそのままビルドして実行できる「完全なコード」にしてください。
*   **使用**: .cppと.hのみの使用で、GUIライブラリや外部ライブラリは一切使用せず、コンソール上で動作するオセロゲームを作成してください。また、mdやjsonなど不要なファイルは作成しないでください。

---

## 2. 画面表示とUI仕様
コンソール上に文字を使ってオセロ盤を描画してください。
*   **盤面サイズ**: 8×8
*   **使用する文字**:
    *   枠線・グリッド: `|`（縦線）や `ー`（横線）
    *   黒石: `●`
    *   白石: `〇`
    *   空きマス: `・`（または半角スペースなど、見やすいもの）
    *   カーソル位置: 現在選択しているマスが視覚的にわかるよう、背景色を変えるか、`[ ]` や `★` などのカーソル専用文字で囲む、あるいは点滅させるなどして強調してください。
*   **描画の更新**: 毎ターン、コンソール画面をクリア（Windows環境なので `system("cls")` 等を許容）し、最新の盤面、現在のプレイヤー（● or 〇）、スコア（お互いの石の数）を綺麗に再描画してください。

---

## 3. 操作方法
キーボード入力のみで操作できるようにしてください（マウス不可）。
*   **移動**: 矢印キー（`↑` `↓` `←` `→`）でカーソルを上下左右に移動。
*   **決定**: `Enter` キー（または `Space` キー）で石を置く。
*   ※ キー入力の検知には Windows 標準の `<conio.h>` の `_getch()` を使用してください。

---

## 4. ゲームロジック
*   **初期配置**: 中央に黒（●）2つ、白（〇）2つを交互に配置してスタート。
*   **ターン遷移**: 黒（●）から開始。交互にターンが移ります。
*   **石の反転ルール**:
    *   選択したマスに石を置いたとき、縦・横・斜めの8方向について、相手の石を挟めるかを正しく判定し、挟んだ石をすべて自分の色に反転させてください。
    *   ルール上、自分の石を置くことで「相手の石を1枚以上ひっくり返せる場所」にしか置けないようにバリデーション（チェック）を行ってください。
*   **パスの判定**:
    *   現在ターンのプレイヤーがどこにも石を置けない場合、自動的に「パス（PASS）」となり、相手のターンに移る処理を入れてください。
*   **ゲーム終了判定**:
    *   両者ともに置く場所がなくなった（盤面がすべて埋まった、または両者パスになった）時点でゲーム終了。
    *   最終的なお互いの石の数を集計し、勝敗（「黒の勝ち」「白の勝ち」「引き分け」）を表示してください。

---

## 5. 出力に関する重要ルール（文字数制限対策）
*   コード全体のボリュームが大きくなり、一度の出力制限（トークン制限）を超える可能性がある場合は、自動的に **[第1部]**, **[第2部]**, **[第3部]** のように分割して出力してください。
*   各部を出力する際は、どこからどこまでをコピペすればよいのか（「ここまでを main.cpp の上部に、次を中部に…」など）を明確に指示し、すべての部を合わせることで**1文字も欠けることなく完全に動く C++ コード**が完成するようにしてください。途中で「省略」や「~~（ロジックは各自実装してください）」といった記述は絶対にしないでください。

## 6. 必ず守るべきルール
*   修正がある場合は必ず、ビルドチェックをしてください。

準備ができたら、コードの出力を開始してください。
